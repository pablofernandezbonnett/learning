# General Refresh Path

Use this path when you want a broad backend refresh rather than a single-topic deep dive.

## Recommended Order

1. Java
2. Kotlin
3. Spring Boot
4. Databases
5. API design
6. Architecture
7. System design
8. Security
9. Cloud and DevOps
10. Testing
11. Algorithms

## Working Rule

For each topic:

1. reopen the smallest mental model
2. review one practical example
3. note the main tradeoff or failure mode
4. stop once the topic feels usable again

This path is for refresh and practical understanding.
Once you finish the broad pass, use each topic folder in this order:

1. `Refresh`
2. `Required`
3. `Growth`
